Explain “repaint” and “reflow” of the webpage.

Reflow is the process of the browser recalculating the layout and position of elements on the page. It happens when changes are made to the elements, that affect the layout of the partial or whole page. Reflow is essentially the computation of the offsets and dimensions of given elements to understand what comes where and takes how much of space, and so on.

Repaint is the process of the browser re-drawing elements on the page. It occurs when changes are made to the appearance of the elements that change the visibility, but doesn’t affect the layout. Repaint is the next stage after reflow. It is when the computation made in the reflow stage is actually used to paint pixels on the screen.

How does JavaScript on the webpage affects the page load speed and performance?

JavaScript has a greater potential for negative performance impact — it can significantly impact download times, rendering performance, and CPU and battery usage. While images and video account for over 70% of the bytes downloaded for the average website, byte per byte.

Each file (CSS, JavaScript, images) on your webpage requires an HTTP request. Minimizing these requests can significantly improve load times.

Putting your scripts at the bottom of the page body lets the browser load the page first. While a script is downloading, the browser will not start any other downloads. In addition all parsing and rendering activity might be blocked.

What is “minification” of the script? Why it is useful to minify your scripts?

A web worker is a JavaScript that runs in the background, independently of other scripts, without affecting the performance of the page. You can continue to do whatever you want: clicking, selecting things, etc., while the web worker runs in the background.

Web Workers are a simple means for web content to run scripts in background threads. The worker thread can perform tasks without interfering with the user interface.
