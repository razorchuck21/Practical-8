What is selector’s specificity and how it’s calculated?

Selector specificity in CSS is a measure of how specific a CSS selector is in the way it targets an HTML element. It is used in the specificity algorithm to determine the final CSS property values for a given element in the DOM tree. Specificity is calculated by starting at 0, adding 100 for each ID value, adding 10 for each class value (or pseudo-class or attribute selector), and adding 1 for each element selector or pseudo-element. Inline style gets a specificity value of 1000, and is always given the highest priority.

What are complex selectors?

Complex selectors in CSS are sequences of one or more simple and/or compound selectors that are separated by combinators. They represent a set of simultaneous conditions on a set of elements. Complex selectors use relationships between the elements as matching criteria. To define this relationship, we use combinators between two identifiers. They can be much more sophisticated and specific than simple selectors.

What is the difference between pseudo-classes and pseudo-elements?

Pseudo-classes and pseudo-elements are two types of keywords in CSS that you can combine with selectors. They are used to target the element's state or specific parts of an element. Pseudo-classes are used to target an element that's in a specific state, such as being hovered over by the mouse pointer. On the other hand, pseudo-elements are used to style specified parts of an element. They can be used to target the first letter or first line, or to insert content before or after the element.
